Why now: Every DeepSeek timeout that returns [] increments triadFailureRate, which governance reads as model quality failure and may trigger athena_soe_recalibration at velocity < 0.7. The system is changing weights because of network latency, not model quality. This is the most damaging active issue.

Snapshot-based gate CI
Commits a frozen set of real LLM outputs as test fixtures. CI runs gates against this snapshot and fails if drop rate drifts outside a tunable band. Turns behavioral quality from "we assume it's fine" into a measurable guardrail.
Why this matters: Current CI (60 Vitest tests) verifies types and contracts but says nothing about whether gates drop 5% or 95% of real candidates. A snapshot CI adds behavioral correctness without API costs in CI. Run after M1 so the snapshot is captured with correct timeouts.

Selection logging — first quality signal
Adds a triad_candidate_selected telemetry event when the user picks a preset or exports it. This is the first signal that distinguishes "valid output" from "output the user actually wanted." No hidden state. No feedback loop. Just auditable data for later offline analysis.
Why this matters: Every gate ensures candidates are valid. Nothing currently tracks which candidates are chosen. Without that signal, you cannot know if your highest-weighted panelist (DEEPSEEK 0.40) actually produces the most preferred output. This event creates that data without any runtime mutation.

### 1. packages/shared-engine/telemetry.ts
Add typed helper function (alongside existing logTriadRunEnd etc.):

  export function logCandidateSelected(payload: {
    runId: string;
    selectedIndex: number;        // 0-based index in the scored candidates array
    selectedPanelist: string;     // e.g. "DEEPSEEK"
    alchemistCodename: string;    // "ATHENA" | "HERMES" | "HESTIA"
    selectionTrigger: 'export' | 'preview' | 'explicit';
    weightedScore: number;        // candidate.score * PANELIST_WEIGHTS[panelist]
    totalCandidates: number;      // how many were shown
    promptLength: number;         // prompt.length (not the prompt itself)
  }): void {
    logEvent('triad_candidate_selected', payload);
  }
