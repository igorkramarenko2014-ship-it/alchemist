## Background
packages/shared-engine/reliability/tablebase-db.ts exports TABLEBASE_RECORDS = []
It is intentionally empty — the system works without it but the keyword
short-circuit path (preset_tablebase_hit) is never exercised.

Real .fxp libraries are now widely available (Serum 2 ecosystem, 2025–2026).
This prompt builds a pipeline to extract, validate, and load real records.

## What to build

### Step 1: scripts/extract-preset-metadata.mjs  (new file)
Input: directory path containing .fxp files (pass as argument or env)
  node scripts/extract-preset-metadata.mjs --dir ~/Downloads/cymatics-free-pack

For each .fxp file:
1. Read raw bytes
2. Extract paramArray using serum-offset-map.ts offsets
   (import from packages/fxp-encoder/serum-offset-map.ts)
   paramArray = 128 floats from the validated offset positions
3. Validate each param value is in [0, 1] via validateSerumParamArray
   from @alchemist/shared-engine/validate
4. Skip file if validation fails — log skipped count at end
5. Extract preset name from .fxp header bytes (offset 0x0C, length 28, ASCII)
6. Build a metadata record:
   {
     "name": string,                // preset name from .fxp header
     "sourcePack": string,          // from --source flag or env PRESET_SOURCE
     "licenseType": "free" | "commercial_personal",  // from --license flag
     "paramArray": number[],        // 128 validated floats
     "extractedAt": ISO string
   }

Write output to: tools/preset-metadata-extracted.jsonl
One record per line (newline-delimited JSON).
logEvent("preset_extraction_complete", { total, validated, skipped, source })

### Step 2: scripts/build-tablebase-records.mjs  (new file)
Input: tools/preset-metadata-extracted.jsonl
Groups presets by musical character using rankTaxonomy from @alchemist/shared-engine

For each preset, infer keyword set from preset name heuristics:
  "bass" → ["bass", "sub", "low"]
  "lead" → ["lead", "synth", "melody"]
  "pad"  → ["pad", "ambient", "texture"]
  "pluck"→ ["pluck", "arp", "short"]
  "fx"   → ["fx", "riser", "impact"]
  default→ ["preset", "serum"]

Builds tablebase records in the schema from tablebase-schema.ts:
  {
    keywords: string[],
    candidates: AICandidate[]   // typed from @alchemist/shared-types
  }
Where each AICandidate has:
  panelist: "DEEPSEEK"          // use DEEPSEEK as source panelist for weightedScore
  score: 0.7                    // conservative default for tablebase entries
  reasoning: " preset from . paramArray validated."
  paramArray: number[]          // from extracted metadata
  state: {}                     // empty SerumState skeleton (serum-offset-map defines it)

Write output to: tools/tablebase-records-generated.json
logEvent("tablebase_build_complete", { recordCount, candidateCount, sources })

### Step 3: Manual review step (NOT automated)
Operator opens tools/tablebase-records-generated.json and reviews:
- Are the keyword groupings sensible?
- Are the reasoning strings legible (≥20 chars — REASONING_LEGIBILITY_MIN_CHARS)?
- Remove any records with suspicious paramArray values
After review, operator copies approved records to:
  packages/shared-engine/reliability/tablebase-db.ts
  (replace the empty TABLEBASE_RECORDS = [] with the approved array)

### Step 4: Add package.json scripts
"preset:extract": "node scripts/extract-preset-metadata.mjs",
"preset:build-tablebase": "node scripts/build-tablebase-records.mjs"

### Step 5: Update .gitignore
tools/preset-metadata-extracted.jsonl   # contains extracted param data
tools/tablebase-records-generated.json  # pre-review draft
