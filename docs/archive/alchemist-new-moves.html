Circuit breaker for repeated panelist failures
When a panelist fails N times in a row, skip it for a cool-down window rather than hammering it and generating repeated empty-array failures that corrupt triadFailureRate and trigger false athena_soe_recalibration. State lives in module memory — no persistent storage, no shadow governance.
Why this is new: The existing timeout queue entry (PANELIST_TIMEOUT_MS) fixes per-call latency. The circuit breaker is different — it handles the case where a provider is down or rate-limited and every call is failing. Without it, a single broken provider forces triadFailureRate to 0.33+ per run, which governance reads as model quality failure and recalibrates weights. The breaker prevents that feedback loop while being fully auditable.

Gate baseline + regression CI check
The existing snapshot CI (in the queue) detects gate failures. This adds a committed tools/gate-baseline.json that encodes the known-good drop rate from your first real calibration run. CI fails if the current run drifts more than 10 percentage points above it — catching silent quality decay before it reaches users.
Why this is new: The snapshot CI checks absolute gate pass/fail. This checks relative drift against a committed baseline. The difference: snapshot CI tells you gates are broken; baseline CI tells you gates are quietly getting worse. Both are needed. Run this after your first successful pnpm test:real-gates with all three keys live.

This file IS committed to git (unlike gate-calibration-output.json which is gitignored).

### 2. scripts/check-gate-regression.mjs  (new file)
- Reads tools/gate-baseline.json (exit with clear error if missing)
- Reads tools/gate-calibration-output.json (exit with clear error if missing)
- Computes drift per gate: current.survivalRates.X - baseline.survivalRates.X
- Fails (exit 1) if:
    any survival rate dropped more than REGRESSION_THRESHOLD (default 0.10) below baseline
    OR reasoningLengths.pctUnder20 > 0 in current (hard — zero tolerance)
- On failure: prints human-readable diff table to stdout
- On pass: prints summary table
- Env: REGRESSION_THRESHOLD overrides default 0.10

### 3. Add to root package.json scripts
"gate:check-regression": "node scripts/check-gate-regression.mjs"

### 4. Usage in release flow (add to docs)

In CI (optional, env flag to disable for initial runs):

SOE hint consumer — stderr → local file
verify_post_summary.soeHint is emitted to stderr after every verify run but currently goes nowhere. This adds a tiny script that wraps the verify commands and appends soeHint lines to a local soe-hints.jsonl file for offline review. One-command viewer included.
Why this is new: SOE hints are currently invisible — they emit to stderr and disappear. Accumulating them in a local append-only file creates an ops history that feeds directly into mergeGreatLibraryIntoSoeSnapshot (FIRE §K) when you're ready to close the feedback loop. Zero impact on production. Dev-only tool, gitignored output.

## What to build

### scripts/verify-with-soe-capture.mjs  (new file)
A thin wrapper around the existing verify commands that captures
verify_post_summary events from stderr and appends them to a local file.

Usage:
  node scripts/verify-with-soe-capture.mjs [harsh|web]
  # Default: 'harsh' (runs pnpm verify:harsh)

Behavior:
1. Spawn: node scripts/with-pnpm.mjs [verify:harsh | harshcheck]
   (reuse existing with-pnpm.mjs — do not reinvent)
2. Pipe stderr line-by-line
3. For each line containing "verify_post_summary":
   - Parse JSON
   - Append to soe-hints.jsonl:
     { "ts": ISO, "mode": ..., "exitCode": ..., "soeHint": ..., "durationMs": ... }
4. Pass all stderr through unchanged (operator still sees normal output)
5. Exit with same code as spawned process

Output file: soe-hints.jsonl at monorepo root (gitignored)
Each line is one JSON object (newline-delimited JSON).

### scripts/soe-review.mjs  (new file)
Reads soe-hints.jsonl and prints a human-readable summary:

  SOE HINT HISTORY (last 20 runs)
  ────────────────────────────────
  2026-03-22T14:01  verify-harsh  exit:0  hint: "triadFailureRate nominal"
  2026-03-22T13:44  verify-harsh  exit:0  hint: "latency within threshold"
  ...

  Most recent soeHint: "..."
  Runs captured: N   Green: N   Failed: N

### Add to root package.json scripts
