Human-supervised calibration loop
Calibration currently produces data but no path to action. This adds a gate:suggest script that reads calibration JSON, computes per-gate sensitivity, and prints a suggestion report. Operator reads it, decides, commits manually. No auto-apply. No runtime mutation.
The rule: pnpm gate:apply that auto-writes constants is forbidden by FIRE §I (no unauditable runtime mutation). The suggestion script is the correct version — it produces a human-readable diff proposal, logs one gate_suggestion_report event to stderr, and stops there. The operator commits the change.
